1. MCP SERVER (название: product-mcp)
2. TELEGRAM BOT

Промпт условно разделён на три части: в первой мы сделаем описание работы сервера на Flask или FastAPI, и базы данных на SQLite3 для работы с MCP сервером. Тут же создадим базовую таблицу для работы. Для примера MCP инструментов зададим работу со списком товаров (добавление элементов, получение списка элементов) и какой-нибудь безопасный калькулятор.

# BOT LOGIC
0. Пользователь выбирает ai-провайдера, модель, температуру, токены из доступных вариантов в скриптах ai_direct.py и openai_client.py.

1) MCP SERVER (название: product-mcp)

Функции сервера:
- использовать SQLite3 локальную базу данных
- создать таблицу products с колонками:
-- id INTEGER PRIMARY KEY AUTOINCREMENT,
-- name TEXT,
-- category TEXT,
-- price REAL
- при первом запуске автоматически заполнить таблицу 100 тестовыми товарами (разные категории, цены, названия)
- реализовать MCP инструменты:
	a) list_products() → возвращает список товаров.
	b) find_product(name: string) → возвращает товары по совпадению имени.
	c) add_product(name: string, category: string, price: float) → добавляет товар.
	d) calculate(expression: string) → безопасный калькулятор (eval запрещён, использовать python AST или простой парсер).

Требования:
- сервер должен быть MCP-совместимым
- экспортировать инструменты в формате MCP JSON schema
- запускаться командой: python server.py

Структура:
mcp_server/
├── server.py
├── db.py
├── tools.py
├── products.db (создаётся автоматически при запуске)
└── requirements.txt

В requirements включить:
- openai
- sqlite3 (если нужно указать)
- flask или fastapi (на выбор)
- python-multipart
- mcp-spec (если требуется)

Во второй части мы опишем работу Telegram-бота для взаимодействия с сервером с LLM-моделью по протоколу HTTP. Здесь мы тоже задаём структуру, используемую модель и логику работы. В третьей части уточним, что нам нужен готовый код с корректным хранением переменных среды в файле .env.

2) TELEGRAM BOT

Бот должен:
- работать через python-telegram-bot или aiogram (любой)
- подключаться к MCP серверу через HTTP
- использовать модель OpenAI GPT-4 / GPT-4o-mini (ChatCompletions API)
- при сообщении пользователя:
		→ бот отправляет текст в LLM.
		→ LLM решает, нужно ли вызвать MCP tool.
		→ если нужно, бот делает запрос к MCP tool и подставляет ответ.
		→ бот отправляет финальный ответ человеку.
		
Поведение:
- Пользователь может писать обычным языком: "покажи все товары", "найди чай", "добавь товар яблоки 120 фрукт".
- Бот отвечает красиво, структурировано.
- Если запрос непонятен → бот задаёт уточняющий вопрос.

Структура:
telegram_bot/
├── bot.py
├── config.py (хранить TELEGRAM_API_TOKEN и OPENAI_API_KEY через .env)
├── mcp_client.py (функции вызова MCP инструментов)
└── requirements.txt

Сейчас сгенерируй весь код проекта полностью.

Структура проекта должна состоять, помимо файлов README и .env, из двух папок:
- mcp_server, где описывается логика работы сервера,
- telegram_bot, где написана логика работы бота.
В mcp_server должна быть логика работы сервера (server.py), логика работы базы данных, её инициализация и подключение к ней, задаваемые файлом db.py. 
В requirements.txt должны быть зависимости от FastAPI (веб-фреймворк), uvicorn (веб-сервер) и pydantic (библиотека для валидации данных). 
В tools.py должныц быть определены безопасный калькулятор safe_eval и методы для работы с MCP. 
Мы специально потребовали, чтобы калькулятор не использовал функцию eval() в питоне, т.к. она может использоваться для выполнения любого кода, даже вредоносного, в терминале сервера, что делает её небезопасной. 

Проверь требования:

1. Инструменты задаются в массиве объектов типа JSON. Для каждого инструмента мы определяем имя name, описание description и inputSchema — структура передаваемых данных (например, для LLM). Например, для list_products нам достаточно только одного объекта для просмотра всего списка продуктов. Для find_product же нам также нужно передавать поисковый запрос в виде строки.

2. В telegram_bot также есть несколько исполняемых файлов. Структура самого бота определяется в bot.py. В mcp_client.py мы задаём обработку и выдачу запросов через бота. Ещё мы имеем файл конфигурации config.py, содержащий в том числе ссылки на токены Telegram и OpenAI.

3. Запускаем под venv файлы mcp_server\server.py, затем telegram_bot\bot.py командамиpython .\server.py и python .\bot.py, находясь в соответствующих папках. Проверяем соответствующие логи в терминале и убеждаемся в том, что у нас запущен сервер и инициализирован бот. Чтобы не менять директории в одном терминале, можно запустить два терминала для двух разных папок.

В файле bot.py мы также задаём промпт, по которому работает бот Telegram. Там же происходит обработка и передача контекста, а также форматирование запросов и ответа.

Ты — умный помощник для работы с базой данных товаров.

Ты можешь использовать следующие инструменты:

list_products - показать все товары
find_product - найти товары по имени (требует параметр "name")
find_products_by_category - найти товары по категории (требует параметр "category")
add_product - добавить товар (требует параметры "name", "category", "price")
calculate - вычислить математическое выражение (требует параметр "expression")
Когда пользователь просит что-то сделать, определи, какой инструмент нужен, и верни JSON в формате:

{
"tool": "название_инструмента",
"arguments": {"параметр": "значение"}
}

Если инструмент не нужен, просто ответь пользователю обычным текстом.

Примеры:

"покажи все товары" → {"tool": "list_products", "arguments": {}}
"найди чай" → {"tool": "find_product", "arguments": {"name": "чай"}}
"покажи товары в категории электроника" → {"tool": "find_products_by_category", "arguments": {"category": "Электроника"}}
"найди все товары категории одежда" → {"tool": "find_products_by_category", "arguments": {"category": "Одежда"}}
"добавь товар яблоки 120 фрукт" → {"tool": "add_product", "arguments": {"name": "яблоки", "category": "фрукт", "price": 120}}
"сколько будет 2+2" → {"tool": "calculate", "arguments": {"expression": "2+2"}}
Отвечай на русском языке, будь дружелюбным и полезным.
